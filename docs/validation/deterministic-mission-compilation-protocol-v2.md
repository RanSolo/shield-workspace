# Deterministic Mission Compilation Corrected Freeze Protocol v2

This protocol creates a new experimental candidate. It does not amend v0 or
failed v1 and their outcomes must remain separate.

## Correction boundary

V1 failed because one digest was transcribed incorrectly into its frozen
manifest. The runtime-envelope specification artifact itself was unchanged and
all earlier identity checks passed. V2 changes only candidate-level and affected
runtime-wrapper, target-profile, runtime-envelope, envelope-specification, and
freeze identities.

Unchanged compiler-side and arm-prompt components retain their immutable
component identities and bytes. V2 independently rehashes and binds them; no v0
or v1 result is imported as a v2 result.

## Derived manifest rule

Every manifest artifact binding is generated by reading the exact artifact
path and computing SHA-256 over its bytes. The generator accepts no expected
artifact hash as input. It records the path, byte length, and derived hash.

An independent verification pass must reread each path, recompute every hash,
verify path uniqueness and lexical order, and compare semantic runtime-envelope
bindings to the exact specification bytes. Missing, extra, reordered,
substituted, malformed, or mismatched bindings fail closed.

The v2 package source-tree digest is independently derived from the sorted list
of package-relative paths, byte lengths, and SHA-256 hashes. The source commit,
dependency lock, runtime versions, complete command matrix, and prior evidence
separation are frozen explicitly.

## Runtime envelope v2

The runtime envelope remains a canonical two-message transport:

1. system bytes are the exact v2 common wrapper immediately followed by the
   unchanged shared-runtime-instruction bytes, with an explicitly empty
   separator;
2. user bytes are the exact arm prompt bytes.

The envelope now binds the v2 experiment, trial, candidate, envelope-spec SHA,
target profile, wrapper, shared instruction set, arm-prompt digest, arm name,
and exact Base64 message bytes. Runtime-added messages remain forbidden.

## Required focused regression

At minimum, tests must prove that:

- changing one nibble of the recorded envelope-specification digest returns
  `RUNTIME_ENVELOPE_SPEC_DIGEST_MISMATCH`;
- changing specification bytes while retaining the digest returns the same
  closed mismatch;
- a malformed digest returns
  `RUNTIME_ENVELOPE_SPEC_DIGEST_MALFORMED`;
- cross-arm substitution and appended messages fail closed; and
- both exact envelopes verify against the directly derived specification hash.

## Freeze and measurement

1. Commit the complete v2 construction source and generated runtime artifacts.
2. Build and validate from a clean Git archive.
3. Generate the freeze manifest from exact committed artifacts.
4. Independently verify every frozen binding before committing the freeze.
5. Commit the freeze manifest.
6. Extract the frozen revision independently and rerun v0, v1, v2, manifest
   verification, frozen identity comparisons, and production validation.
7. Record results separately and return the exact evidence revision to Fury.

Any post-freeze change to an artifact, identity, dependency, protocol, fixture,
isolation contract, rubric, wrapper, prompt, or envelope invalidates the run.
No model invocation is part of Stage A.
